Welcome to our API Documentation.

In order to get started its as easy as:
  1. Downloading node package manage(npm)
  2. Opening your Command Shell/Terminal in the directory that contains our files
  3. Typing in the command npm start
    • If you are missing some of the required packages you may need to use npm install before doing this.
    • See here for a list of these packages.
It is our hope that this document helps you to easily navigate our website + server and give you full functionality to our DATABASE.

Once you have everything setup click here to visit our webshop!

This is our guide to using our HTTP1.1 methods
Methods Actions
GET This will retrieve all data from our database and return in an array of json elements with key value pairs
POST Once filling in our form, using the submit button, you will have logged a POST request which adds the new elements to our database in a json string. Each entry is given a unique value for the ID key.
PUT Use the unique id KEY to update elements
DELETE Use brand name to delete elements
This are our URL paths
Path Description
'http://localhost:3000' use this URL you can get our all phones' information
'/phones' use this path you can GET or POST phone information
'/update' use this path to change resources
'/delete' use this path to delete elements

A Few Common Errors

The screensize must be an numerical value. Either a float or an int, anything else will cause an error
The image block requires a url or else nothing will be rendered.
The database is stored in JSON fromat and the form data is parsed the same way.
Attempting to submit form data to our database externally, without the proper formatting, will cause errors

response status
STATUS CODE Meaning
200 Standard response for successful HTTP requests.
202 The request has been accepted for processing, but the processing has not been completed
404 The requested resource could not be found but may be available in the future
500 Server side error and should be reported to the owner of the website
